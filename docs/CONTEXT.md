# CONTEXT.md

## Scope
- `docs/`는 저장소 자체 운영과 릴리즈 절차를 설명하는 문서를 둔다.
- 자세한 실행 규칙은 [../CLAUDE.md](../CLAUDE.md)와 함께 읽는다.

## Key Files
- [release-checklist.md](./release-checklist.md): 태그 생성 전후의 운영 체크리스트
- [release-notes-template.md](./release-notes-template.md): GitHub Release 초안 템플릿

## Local Rules
- 문서는 운영자가 바로 실행할 수 있는 실제 파일 경로와 명령만 적기.
- 릴리즈 문서를 바꿀 때는 `README.md`와 `CHANGELOG.md`의 링크 및 흐름도 함께 확인하기.
- 장문 배경 설명보다 절차와 검증 포인트를 우선 적기.

## Verification Notes
- 릴리즈 문서가 바뀌면 링크 경로와 명령 예시를 다시 점검하기.
- 문서 구조가 커지면 하위 문서로 분리하고 여기서는 진입점만 유지하기.
